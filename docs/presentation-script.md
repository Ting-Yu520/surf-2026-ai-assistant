# 教授演示脚本 · SURF-2026-0154

## 演示前准备

1. 确保 `.env` 文件中有你的 DeepSeek API Key
2. 终端运行：`streamlit run src/app.py`
3. 浏览器打开后，确认左侧 JSON 数据已加载、场景可选
4. **提前测试一次**：选 "The Impossible Pass" → 点生成 → 确认输出质量没问题

---

## 演示流程（全程约 5 分钟）

### 第一步：打开页面（30 秒）—— 建立问题

**你要说的：**

> "老师，我现在想跟您汇报一下我对于 SURF 项目的初步想法和技术验证。"

> "我的核心问题是：**现在的 AI 体育分析系统，全部是为专业教练和资深球迷设计的。** 一个完全不懂足球、甚至对足球没兴趣的人，面对这些数据，什么都看不懂。"

**你操作：** 指着左侧的 JSON 数据。

> "您看左边这些数据——坐标、概率、传球通道宽度——这是一个计算机视觉系统从比赛视频里提取出来的。对普通人来说，这就是一堆数字，没有任何意义。"

---

### 第二步：生成解说（1 分钟）—— 展示方案

**你要说的：**

> "我做的事情是：**用 Prompt Engineering 做核心算法，让 LLM 把这些数据'翻译'成任何人都能理解的故事。**"

> "我现在点这个按钮，AI 会读取左边的 JSON，生成一段文字——但它的目标读者不是教练，而是**一个这辈子第一次看足球的人**。"

**你操作：** 点击"生成通俗解说"按钮。等待 10-20 秒。右侧出现生成的故事。

> "您看右边的输出。它没有用任何足球术语。它说的是：'这就像你去厨房，想从冰箱里拿瓶水，但家里三个小孩刚好把过道堵得严严实实的感觉。' —— 这是生活化的类比，不需要任何足球知识就能理解。"

**关键点（指着具体段落）：**

> "注意这段——它把 0.8 米的传球通道翻译成了'一个足球两边各剩两厘米，比你的大拇指宽不了多少'。把 5.2% 的成功率变成了'像灌了铅的腿，稍微控制不好就被断走'。**这才是 AI 应该做的事情——不是扔一堆数字给你，而是让你感受到这个瞬间的紧张和不可能。**"

---

### 第三步：解释技术路线（1 分钟）—— 展现学术判断

**你要说的：**

> "为什么我不做预测模型？因为我做了技术边界评估。"

> "DeepMind 的 TacticAI 能做角球预测，是因为他们有利物浦俱乐部独家提供的多机位 3D 追踪数据。我们只能拿到公开的电视转播画面——单视角、有遮挡、镜头会切。用这种数据做预测，准确率达不到学术标准。"

> "所以我做的是 **Generative HCI for Sports Analytics**——不跟 DeepMind 拼底层算法，而是用他们做不到的方式，解决他们没解决的问题：**让体育数据对普通人变得有意义。**"

---

### 第四步：长期规划（1 分钟）—— 展现野心和可行性

**你要说的：**

> "我现在的 Demo 用的是手工构造的 JSON 来模拟 CV 输出。这是 Phase 1 的概念验证——证明这条数据流走得通。"

> "SURF 的 10 周内，我可以把这个流程完善成：**真实比赛视频 → VLM 识别关键帧 → 自动提取战术 JSON → LLM 生成新手友好叙事。** 10 周结束时，交付一个完整的 Working Prototype。"

> "到了大四 FYP，核心工作就不是开发了，而是**严谨的用户研究**。我会招募至少 30 名不看足球的受试者，做 A/B 测试——一组看传统解说，一组看 AI 生成的解说——用 NASA-TLX 认知负荷量表和统计学方法，证明这套系统真的降低了理解门槛。"

> "这恰恰对应了您 SURF 项目描述里的 'Evaluating AI recommendations with real users'。"

---

### 第五步：收尾（30 秒）—— 提出请求

**你要说的：**

> "最终目标是在 HCI 或 Applied AI 方向发表一篇高水平论文——ACM CHI、IUI 或 DIS 这个级别。论文的叙事很清晰：**现有 AI 体育系统面向专业用户，我们证明了 AI 可以把专业数据转化为零基础受众可理解的体验，并用量化数据验证了效果。**"

> "如果我的表现能达到您的要求，我非常希望邀请您担任我的 FYP 导师，把这个 SURF 项目无缝延伸为我的毕业设计。"

## 可能的教授追问 & 你的回答

### Q: "你这个和 ChatGPT 有什么区别？不是直接用大模型吗？"
**A:** "区别在于 Prompt Engineering 的深度。普通的 ChatGPT 你问它'解释这个战术'它会给出定义式的回答。我的 System Prompt 经过了严格设计——禁止术语、强制生活类比、要求基于数据而非编造——这是一套**约束系统**，确保输出质量的一致性。这个 Prompt 本身就是我的核心 IP。"

### Q: "你怎么证明你的解释比传统解说更好？"
**A:** "这就是为什么需要 FYP 阶段的用户研究。不做用户研究的话，'更好'只是我的主观判断。我要用 NASA-TLX 量表测认知负荷、用 Likert 量表测理解度和兴趣度、用独立样本 T 检验比较两组——统计学显著的结果才是学术证据。"

### Q: "为什么不直接训练一个模型？"
**A:** "训练模型需要大量标注数据，而且 10 周内训练不出可靠的模型。更重要的是——如果我的优势是系统分析和 Prompt 工程，那我的论文应该发在 HCI 领域，而不是 ML 领域。HCI 看重的是 AI 系统如何影响用户体验，不要求你发明新算法。这是扬长避短。"

### Q: "10 周能做完吗？"
**A:** "能。因为我做的不是从零训练模型，而是集成现有工具——VLM 是现成的 API，LLM 也是现成的 API，我的核心工作是 Prompt Engineering 和系统集成。10 周足够从概念验证做到完整原型。"
