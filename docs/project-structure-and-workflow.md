## 一、一句话说清项目在做什么

> **用 AI 把足球战术数据"翻译"成完全不懂足球的人也能听懂的科普解说。**
> 形式：双口相声风格（一个小白问、一个懂哥答）+ 中文配音 + 视频合成。

## 三、管线工作流程（这是要给老师讲的重点）

### 3.1 整体流通图

```
                        用户操作：选角球 → 点生成
                              │
                              ▼
┌────────────────── STEP 1 ──────────────────────┐
│  phase_bridge.py                                │
│  输入：corner_kicks_2026.json 中某一条角球数据   │
│        + corner_articles.json 中对应的央视新闻   │
│  处理：把原始数据拆成两段                        │
│        → fact_section（比赛基本信息）            │
│        → tactic_section（战术彩蛋数据）          │
│  输出：结构化的两段文本                          │
└────────────────────┬────────────────────────────┘
                     │
                     ▼
┌────────────────── STEP 2 ──────────────────────┐
│  prompts/corner_kick.py                         │
│  输入：fact_section + tactic_section            │
│  处理：套进 System Prompt + User Template       │
│        → 发给 DeepSeek V4 Pro LLM               │
│  输出：双口相声脚本（A懂哥+B小白，4-6轮对话）    │
│                                                 │
│  ★ 核心 IP 就在这：System Prompt 的约束系统      │
│    · 角色约束：A是懂哥（红）、B是小白（蓝）       │
│    · 术语禁令：只能用生活比喻，不能说专业术语     │
│    · 数据约束：只能基于提供的数据，禁止编造       │
│    · 格式约束：输出##VISUAL##指令给视频用        │
└────────────────────┬────────────────────────────┘
                     │
                     ▼
┌────────────────── STEP 3 ──────────────────────┐
│  video_overlay.py → parse_script()              │
│  输入：LLM 输出的原始脚本字符串                  │
│        "A: 你看那个角球...                       │
│         ##VISUAL## highlight pos=(65,40)        │
│         B: 啥角球？..."                         │
│  处理：按前缀解析每行（A/B/##VISUAL##）          │
│  输出：[{speaker, text, visual}, ...]           │
│        结构化对话段列表                          │
└────────────────────┬────────────────────────────┘
                     │
                     ▼
┌────────────────── STEP 4 ──────────────────────┐
│  tts_client.py                                  │
│  输入：[{narration: "对话文本"}, ...]            │
│  处理：每句话 → 调用 Edge TTS → 生成MP3        │
│         → ffprobe 测出每段精确时长              │
│         → 最后把全部 MP3 合并成一个完整音频      │
│  输出：合并后的 audio.mp3 + 每段时长数据          │
└────────────────────┬────────────────────────────┘
                     │
                     ▼
┌────────────────── STEP 4b ─────────────────────┐
│  video_overlay.py → build_timeline()            │
│  输入：对话段列表 + 每段时长                     │
│  处理：计算每句在什么时间开始/结束               │
│  输出：[{speaker, start: 0.0, end: 3.2}, ...]   │
└────────────────────┬────────────────────────────┘
                     │
                     ▼
┌────────────────── STEP 5 ──────────────────────┐
│  video_overlay.py → create_titled_video()       │
│  输入：原始视频 + 配音音频 + 时间轴              │
│  处理：ffmpeg 两步合成                          │
│        Step A: 叠加滤镜到视频                   │
│          · zoompan → Ken Burns推拉运镜          │
│          · drawtext → 标题卡/尾卡/角标           │
│          · drawbox → 红蓝四边边框（A=红/B=蓝）  │
│          · 高亮圈 → 战术位置红色标记            │
│        Step B: 替换原音频为 AI 配音             │
│  输出：最终解说视频 .mp4                        │

                     ▼

## 四、每个模块详细说明（老师追问细节时用）

### 4.1 config.py — 配置文件（42 行）

**作用：** 存所有 API Key 和模型参数。改了这里就等于改了整个系统的模型后端。

| 参数 | 值 | 说明 |
|------|----|------|
| DeepSeek 模型 | `deepseek-v4-pro` | 主力 LLM，负责生成解说文本 |
| 连接方式 | `api.deepseek.com/anthropic` | 用 Anthropic SDK 连 DeepSeek |
| Gemini 模型 | `gemini-2.5-flash` | 预留的 VLM（视频理解），还没接上 |
| TTS 音色 | `zh-CN-XiaoxiaoNeural` | 微软 Edge TTS 中文女声 |
| TTS 语速 | `+10%` | 比正常稍快，适合科普节奏 |

### 4.2 corner_kick.py — 核心 IP（115 行）

**为什么这是核心 IP？**

这不仅仅是一条 Prompt——它是一个**多维度约束系统**：

| 约束维度 | 具体规则 | 为什么重要 |
|---------|---------|-----------|
| **角色设定** | A=懂哥（红），B=小白（蓝），对抗式对话 | 让完全不懂的人也能代入 B 的视角 |
| **术语禁令** | "角球"="台球自由球"；"禁区"="球门前危险区" | 确保零基础能听懂 |
| **生活比喻** | 每个战术要点必须配一条生活类比 | 把抽象足球战术变成日常场景 |
| **数据约束** | 只能基于提供的 JSON 数据说话 | 防 AI 幻觉（编造不存在的战术） |
| **格式约束** | 输出固定格式：A/B 行 + ##VISUAL## 指令 | 确保后续视频合成能解析 |

**实际输出效果对比：**

> ❌ 传统解说："荷兰队利用角球战术，van Hecke 在后点头球攻门得分，防守球员站位失误。"
>
> ✅ AI 解说："这就像你排队买奶茶的时候假装要走，骗得前面的人一愣神，然后突然冲回去——球已经进了！"

### 4.3 pipeline.py — 总管管线（139 行）

**这就是整个系统的"司令塔"。** `process_corner_kick()` 一个函数走完：

1. 拿到原始数据（JSON 或网页抓取）
2. 发给 DeepSeek LLM → 收到双口相声脚本
3. 解析脚本为逐句对话
4. 逐句 TTS → 生成 MP3 配音
5. 计算时间轴（每句什么时候开始/结束）
6. ffmpeg 合成视频

**输入 → 输出：**
- 输入：一条角球数据（含央视新闻文本）
- 输出：dict 含 script / audio_path / timeline / output_video / elapsed（耗时）

### 4.4 phase_bridge.py — 数据翻译官（162 行）

**作用：** 把"人写的文章"变成"LLM 爱读的结构化数据"。

具体来说，它把一条角球数据拆成 LLM Prompt 需要的两段式：

```
fact_section（事实部分）:
  比赛：日本 vs 荷兰
  时间：88'
  进球者：镰田大地
  战术描述：日本队在身高劣势下，依靠角球配合扳平比分

tactic_section（战术彩蛋）:
  攻击球员：6人
  防守球员：6人
  最可能接球概率：35%
  防守方关键位置：(70, 38)
```

另外 `sample_tacticai_output()` 函数模拟了 TacticAI 的分析输出——虽然现在还是模拟的，但数据会根据每个角球的 `corner_type`（左/右侧角球）不同而变化，不再是固定值。

### 4.5 tts_client.py — 语音合成（99 行）

**用 Edge TTS（免费）** 实现中文配音。

关键设计：
- **逐句生成**：每句话单独生成 MP3，而不是整段读
- **ffprobe 测时长**：精确知道每句话说了几秒
- **失败兜底**：如果哪句 TTS 失败，自动生成静音占位，管线不断
- **FFmpeg 合并**：把逐句 MP3 快速拼接成一个完整音频

### 4.6 video_overlay.py — 视频合成（308 行）

**技术上最复杂的模块。** 用 ffmpeg 在原始比赛视频上叠加多层效果：

```
原始视频
    │
    ├─ zoompan（Ken Burns 推拉运镜——让画面有动感）
    ├─ drawtext（标题卡、尾卡、字幕）
    ├─ drawbox（红蓝彩色边框——A说话边框红、B说话边框蓝）
    └─ drawtext（高亮圈——战术位置红色●标记）
    │
    ▼
最终视频 + TTS 配音音频
```

**为什么不用 moviepy 而用 ffmpeg？**
- moviepy 在 Windows 上有盘符冒号跟 filter 语法冲突的问题
- ffmpeg 更稳定、更快、产出文件更小

### 4.7 app.py — Streamlit 界面（190 行）

```
┌── 顶部：饼图式管线流程 ──────────────────────────┐
│  📹 比赛视频 → 📊 TacticAI分析 → 🎤 二人转 → 📺 合成 │
└───────────────────────────────────────────────────┘
                      │
   ┌──────────────────┴──────────────────┐
   │                                     │
   ▼                                     ▼
┌─ 左栏：Phase 1 数据 ──┐  ┌─ 右栏：Phase 2 生成 ──┐
│  · 角球选择下拉框       │  │  · "生成"按钮          │
│  · 原始 JSON 数据      │  │  · 最终视频播放器       │

## 六、技术决策理由（老师可能会问"为什么用这个"）

| 问题 | 答案 |
|------|------|
| **为什么用 DeepSeek 不是 GPT？** | DeepSeek V4 中文能力极强、性价比高，Anthropic SDK 兼容，切换成本近乎零 |
| **为什么用 Edge TTS 不是商用 TTS？** | 免费、中文自然、支持 SSML 控制语速 |
| **为什么用 ffmpeg 不是 moviepy？** | Windows 盘符冒号与 moviepy 底层语法冲突；ffmpeg 更稳定 |
| **为什么 TacticAI 用模拟数据？** | 真实 TacticAI 需要利物浦级多机位 3D 数据，公开转播做不到。模拟数据已足够验证 LLM 翻译层的效果 |
| **为什么不做视频理解（VLM）？** | 计划中（SURF 下一步）。当前重点是先验证"翻译层"本身有效 |
| **为什么选角球？** | 角球是足球里最频繁的定位球战术，1-2 分钟一个完整叙事，适合做 prototype 验证 |

## 七、技术栈总览

```
LLM：    DeepSeek V4 Pro（通过 Anthropic SDK 连接）
TTS：    微软 Edge TTS（edge-tts Python 库，免费）
视频：   ffmpeg（命令行工具，Zoompan/Drawtext/Drawbox 滤镜）
界面：   Streamlit（Python 交互式 Web 框架）
数据：   JSON 数据集（手动整理）+ CCTV 新闻文本
字体：   微软雅黑（msyh.ttc，中文字幕兼容性最好）
```
